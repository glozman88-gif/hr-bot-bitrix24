# HR-бот для открытых линий Битрикс24 — контекст проекта и продолжение работы

Этот файл — точка входа для AI-агента (Claude Code) на новом компьютере: прочитай его,
подключись к ресурсам и продолжи с места остановки. Реальные ключи — в отдельном
защищённом сообщении (НЕ в этом публичном репозитории).

## Что это за проект

Чат-бот, который общается с соискателями в **открытых линиях Битрикс24**, помнит контекст
последних 30 сообщений, подбирает вакансии (парсятся из кабинета **Авито** через официальный
API), собирает первичную анкету и пишет её в **сделку CRM**, привязанную к чату. Есть
**админ-приложение** в Битрикс24 (placement в левом меню): настройки бота, промпты под разные
задачи, управление вакансиями, соискатели, учёт токенов и **биллинг** (баланс, выписка, счета,
счёт-оферта PDF, обещанные платежи).

Разработчик/владелец: интегратор **Lidkom.ru** (Глозман Е. М.), портал **gkfs.bitrix24.ru**.
Платформа хостинга/деплоя — **VibeCode** (vibecode.bitrix24.tech): серверы Black Hole, AI Router,
прокси Bitrix24 REST, боты, OAuth-приложения. Документация API: `GET /v1/guide` с ключом.

## Архитектура

```
Соискатель → Открытая линия Б24 → бот hr_openline_bot (welcome-бот линии)
   ↕ polling /v1/bots/<BOT_ID>/events каждые 3с
Backend (Express, Node 20 ESM, /opt/app) на BlackHole-сервере VibeCode
   ├─ src/bot/poller.js       — приём событий, дедуп, выход из сессии при нуле баланса
   ├─ src/bot/conversation.js — контекст 30 сообщений → AI Router → JSON {reply,extracted,vacancies}; списание токенов
   ├─ src/bot/crm.js          — сделка берётся из chat.entityData2 (DEAL|id), запись анкеты в comments
   ├─ src/avito/*             — официальный API Авито (core/v1/items + job/v2/vacancies), синхрон 6ч
   ├─ src/billing.js          — баланс в токенах, выписка, счета, обещанные платежи, тариф 1₽/1000
   ├─ billing/invoice.cjs     — PDF счёт-оферта (pdfkit+qrcode, ГОСТ-QR), offer-text.cjs — текст оферты
   ├─ src/admin/api.js + auth.js — REST админки + аутентификация
   └─ public/                 — SPA (vanilla, без сборки): Дашборд, Вакансии, Промпты, Соискатели, Токены, Баланс, Настройки
PostgreSQL (на том же сервере) — settings, prompts, vacancies, messages, candidates,
   conversations, token_usage, billing_account, billing_transactions, invoices, bot_state
AI Router — бесплатный BitrixGPT (модель скрыта из UI); переключение на платную/BYOK — в коде/настройках
```

## Ключевые нюансы платформы (важно!)

- **Передеплой:** рантайм `node20-pg` НЕ идемпотентен — на повторном `/deploy` НЕ передавать `runtime`
  (пакеты уже стоят), иначе падает на создании БД `app`. Рантайм — только на ПЕРВОМ деплое.
- **Открытые линии:** в событии `ONIMBOTV2MESSAGEADD` `data.dialogId` ПУСТОЙ — берём из
  `data.chat.dialogId`/`chat<id>`. Привязанная сделка — в `chat.entityData2` (`...DEAL|<id>`).
  Бот подключается как **welcome-чат-бот линии** (Контакт-центр), не «оператором».
- **Авторизация админки:** браузер режет сторонние cookie в iframe Б24 → не полагаемся на gateway-cookie.
  Сервер вшивает токен сессии в HTML (`window.__VIBE_SESSION`), SPA шлёт его в заголовке `X-App-Session`.
- **Entity API (запись):** поля на ВЕРХНЕМ уровне camelCase (`{title, comments, contactId}`), без обёртки `{fields:{}}`.
- **Авито:** прямой парсинг страниц заблокирован (IP), работает ТОЛЬКО официальный API (client_id/secret).
- **Бот замолчал:** `POST /v1/bots/<BOT_ID>/resubscribe` (подписка на события Б24 протухает).

## ГДЕ ОСТАНОВИЛИСЬ — следующий шаг

**Подключить мониторинг оплаты счетов через банк.** Нужно одно из двух (уточнить у владельца):
1. **Отдельный сервер банковской интеграции** — URL/API + авторизация + эндпоинт статуса оплат; ИЛИ
2. **T-Bank Business OpenAPI напрямую** — `TBANK_TOKEN` + номер счёта; опрашивать
   `GET /openapi/api/v1/bank-statement`, сопоставлять входящие (сумма + ИНН плательщика) с
   неоплаченными счетами → отмечать оплаченными (включая закрытие обещанных платежей).
   Референс готовой реализации: репозиторий `doctorgrunt-app`, `billing/tbank-statement.js`.

Также напомнить владельцу: в настройках линии «Тест чатбота» поставить действие после
чат-бота → «перевести на очередь операторов» (сейчас «закрыть»), чтобы пауза по балансу
корректно передавала диалог сотрудникам.
