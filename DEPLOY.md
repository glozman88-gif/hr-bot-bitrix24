# HR-бот для открытых линий Битрикс24 — развёртывание и эксплуатация

Бот общается с соискателями в открытых линиях, помнит контекст последних 30 сообщений,
подбирает вакансии (парсятся с Авито, рекрутёр галочками выбирает что предлагать),
собирает первичную информацию и обновляет сделку в CRM. Админ-приложение Битрикс24
позволяет настраивать бота, промпты под разные задачи и базу вакансий.

## Архитектура

```
Соискатель → Открытая линия Б24 → бот hr_openline_bot
   ↕ (polling /v1/bots/92473/events каждые 3с)
Backend (Express, /opt/app) на BlackHole-сервере
   ├─ poller        — принимает сообщения, дедуп по message.id
   ├─ conversation  — контекст 30 сообщений из PG → AI Router → JSON {reply, extracted, suggested_vacancy_ids}
   ├─ crm           — резолв сделки (кэш→телефон/контакт→автосоздание) + запись анкеты
   ├─ avito         — парсинг вакансий по расписанию (6ч) + ручной запуск
   └─ admin API+SPA — настройки/промпты/вакансии/соискатели (BFF-auth через gateway)
PostgreSQL — settings, prompts, vacancies, messages, candidates, conversations
AI Router — по умолчанию бесплатный BitrixGPT (model не передаётся); платная/BYOK настраивается в админке
```

## Как пользоваться админкой

**Вариант A (владелец, сразу):** открыть https://app-215a929ee824.vibecode.bitrix24.tech →
кнопка «Войти через Bitrix24» (вход в VibeCode) → откроется панель настройки.

**Вариант B (встроить в левое меню Б24 для сотрудников):**
1. В админке → вкладка «Настройки» → блок «Встраивание в Битрикс24» → ссылка
   «Авторизовать приложение» (или открыть `GET /v1/oauth/authorize?app_key=...&state=...`).
2. Нажать «Опубликовать в левом меню» — приложение появится в левом меню портала
   (`POST /v1/apps/<appId>/publish { placements: ["LEFT_MENU"] }`).

## Что должен сделать администратор на портале

1. **Подключить бота к открытой линии.** В Битрикс24 → Контакт-центр → Открытые линии →
   в нужной линии добавить бота «HR-ассистент» (id 92473) как автоответчик/в очередь.
   Без этого бот не будет получать сообщения соискателей.
2. **Настроить источники Авито.** Админка → Настройки → «Источники вакансий Авито»:
   вставить URL поиска/профиля. Затем вкладка «Вакансии» → «Парсить Авито сейчас».
3. **Отметить вакансии галочками** «Предлагать» — бот предлагает только отмеченные.
4. **Промпты** — вкладка «Промпты»: отредактировать системный промпт скрининга или
   добавить новые типы задач; выбрать активный.

## Известные ограничения / заметки

- **Авито: только официальный API** (`api.avito.ru`, grant `client_credentials`).
  Прямой парсинг страниц кабинета НЕ работает — Авито блокирует все дата-центровые IP
  (HTTP 429/200 «Доступ ограничен» + captcha). Ключи `client_id`/`client_secret` берутся
  в кабинете (Профиль → Настройки → API) и вводятся в админке. Код — `src/avito/avito-api.js`.
  URL-скрейп (`src/avito/parser.js`) оставлен фолбэком, но в проде нерабочий.
- **Бот замолчал?** Подписка на события Б24 со временем «протухает» —
  `POST /v1/bots/92473/resubscribe` (не перерегистрировать бота — это рушит openline-привязки).
- **Сессия админки** держится «пульсом» SPA (пинг `/api/me` каждые 40с); cookie `_vibe_gw`
  живёт 10 мин со sliding-refresh при активности.
- **Связка «открытая линия → сделка»** реализована эвристически (кэш → поиск по
  телефону → автосоздание сделки+контакта), т.к. прямой обёртки сессия→сделка в API нет.
  Уточняется на живых диалогах. Тип сущности (сделка/лид) и автообновление — в Настройках.
- **Black Hole / gateway** при таймауте может отдать HTML со статусом 200 — клиент VibeCode
  это детектит (`GATEWAY_HTML`) и считает временной ошибкой.
- **Публикация placement** требует коммерческого тарифа Б24 (портал `ru_ent250` — ок) и
  предварительной авторизации приложения пользователем (иначе `NO_USER_TOKEN`).
